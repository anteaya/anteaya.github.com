Friday May 20, 2011

I did the Roman Numerals kata this morning. My code is here.

I spent some time reading The Ruby Way this week and have been thinking about the role of specs and mocks in testing. I encountered a situation this week where I created a method that returned two objects and rspec was unable to create a corresponding object for the test. Since I had to also refactor my code substantially (I had over loaded my methods and gotten mired by the result), I have not yet gotten back to creating an appropriate test for that particular method. Paul Pagel shared a link with the apprentices for a Martin Fowler post that discussed the difference between mocks and stubs. I found the differentiation useful. Sometimes I want to test state and sometimes I want to test behaviour and I am glad to know that it is possible to test either.

I like doing katas in the morning. Sometimes I have original thoughts and sometimes I just follow the bouncing ball. I would feel that the second approach is unworthy except that Zed Shaw, whom I feel has some great points to share, has created an entire manual for learning Python on the principal of copying. So watching a video and copying the approach can be a learning experience as well. In any case, both kata and blog post are published so there is something to be said for that.

Happy Friday. If you are one of my friends who get raptured on the weekend, good journey. I guess I'll see the rest of you next week.

Cheers.

Wednesday May 4, 2011

There is a great vid of a Regular Expressions talk by Aaron Kalin from Chicago Ruby on Vimeo. He is writing a gem called graffiti.

He pointed out some good resources during the talk. rubyxp.com is a regular expression tester for ruby 1.9. He suggests the oniguruma gem for use with 1.8. (This requires the libonig-dev library if you don't already have it, I didn't.)

He also suggests a website dedicated to regular expressions, regular-expressions.info.

Mostly just making notes.

One of my apprenticeship assignments is to write a web server. I almost said build. Currently I'm reading up on wikpedia's offering: Internet_socket which is a helpful place to begin. I also found a PDF from PragProg that used to be included in the Pickaxe: app_socket.pdf. Here is a golden oldie from IBM published in 2005 that walks you through sockets: rubysocks.pdf.

I found a short tutorial on creating a simple client and server in Ruby. I put my code here: sockets.
